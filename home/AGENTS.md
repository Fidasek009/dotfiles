# System Notes

This file is for durable context that future agents should know before changing this machine. Keep it short and practical.

Update this file whenever a lasting system preference, config location, or management decision changes. Store only durable context that will still matter in a week or a month and is not obvious from standard inspection.

## System Context

- OS: CachyOS
- Compositor: Hyprland
- Login shell: fish (`/bin/fish`)
- Dotfiles repo: `~/dotfiles`

## Preferences

- Prefer dotfiles for user-level configuration.
- For managed user config, edit files in `~/dotfiles/home/` and keep the live files as symlinks into that repo.
- Keep dotfiles focused on portable preferences; put hardware-specific facts such as Hyprland monitor names, resolutions, positions, scale, and device function-key mappings in machine-local includes.
- Prefer Flatpak for desktop apps.
- Keep system-wide `/etc` edits minimal; use them only when user-level config is insufficient.
- SDDM cannot use the private home wallpaper symlink; keep its wallpaper as a root-readable dotfiles system payload and install it through the dotfiles SDDM installer.
- Do not run `sudo` commands directly; ask the user to run required privileged commands manually.
- Prefer dark UI themes.
- Keep the Czech keyboard layout as the typing default.
- Prefer fail-fast commands over fallback-heavy wrappers unless graceful degradation is clearly needed.
- Never manually edit autogenerated files; use the generator/refresh script that owns them instead.
- Do not inline long shell commands, scripts, or complex config logic into configs; create a purpose-specific script/config file. Avoid one-command wrapper files unless they replace repeated config noise or provide a clear interface.
- Make small, reversible changes and preserve unrelated user edits.
